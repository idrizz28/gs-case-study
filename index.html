The System

Lead in. Appointment out.

Every gym gets Emma — a front desk AI persona that handles inbound SMS leads from Meta ads, Google, and GHL automations.

1
Lead opts in
Ad click or GHL tag fires. Emma engages within seconds.
2
Qualify
Adult vs youth, program interest, single vs multi-enrollee.
3
Capture
Name, email, phone, DOB — all written to GHL in real time.
4
Book
Trial class booked directly into GHL calendar. Confirmation sent.
5
Tag + track
Pipeline tags applied at every stage. Full visibility in CRM.

               jump right in before we talk membership.
12:49 AM LEAD: Yes just for myself
12:49 AM BOT:   Perfect! Let's get you scheduled for a free trial class.
12:52 AM LEAD: dalemoody59@yahoo.com
12:54 AM BOT:   I've got openings tomorrow Friday at 6:30 PM, Tuesday the 26th
               at 6:30 PM, or Wednesday the 27th at 6:30 PM. What works best?

12:56 AM BOT:   ✓ APPOINTMENT BOOKED Dale Moody — Free Trial Class
               You're all set for Wednesday, May 27th at 6:30 PM.

Edge Case Coverage

Built for the hard cases.

Every scenario that would normally drop a lead or require staff intervention is handled in-flow.

👤
New adult lead
Full qualification → data capture → calendar booking, any time of day.
👦
Parent enrolling a child
Youth path with age-routing to the correct kids calendar. Parent DOB also captured.
👨‍👦
Parent + self enrolling
Multi-enrollee: both bookings completed in a single conversation session.
🚫
Under-18 self-booking
Minor gate fires → captures info → routes to team for guardian follow-up. No unsupervised booking.
💬
Pricing question
Firm redirect to trial class every time. No dollar figures given. Ever.
📋
Non-bookable program
Acknowledged without booking path. Redirected to trial for in-person context.
🤝
Staff takes over
ai off tag applies autoShutoff instantly. Bot never re-engages.
⚡
Hostile / opt-out
Graceful de-escalation. No booking attempt made. No argument.
Build Pipeline

Every bot ships the same way.

Three-phase process — plan, build, test. No gym goes live without clearing all quality gates.

Phase 01
/closebot-plan
Structured intake: live GHL calendar pull, program mapping, full flow design. Outputs a JSON spec that feeds directly into the build.
GHL calendar pull flow mapping spec JSON
Phase 02
/closebot-build
KDL generation from Agent Node template, architecture assertion, import + publish via CloseBot API, sandbox attach. No prod until test passes.
Agent Node only API deploy sandbox first
Phase 03
/closebot-test
13-persona eval suite across adult, youth, multi-enrollee, minor gate, pricing, hostile, and non-bookable scenarios. Every booking verified in GHL — not just the transcript.
